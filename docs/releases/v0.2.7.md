## Summary

- Fixes the reviewed reliability and security findings across collection import, k6 run handling, release packaging, and Linux install flows.
- Hardens the APT repository setup installer by verifying the signed SHA256 sidecar with the actual GPG `VALIDSIG` signer and rejecting unexpected keyring primary keys.
- Corrects structured Postman query/form variable encoding so generated k6 scripts use occurrence metadata and RFC3986-compatible percent encoding.
- Adds backend run IDs for k6 metrics, completion, and error events so the UI ignores stale events from previous runs.
- Tightens runner validation, smoke-test bounds, private k6 temp artifacts, threshold parsing, and release asset checks.

## Linux APT Install

Enable the Load Rift APT repository once:

```bash
bash <(curl -fsSL https://github.com/ddv1982/load-rift/releases/latest/download/install-apt-repo.sh)
```

Then install or update normally:

```bash
sudo apt update
sudo apt install load-rift
```

The setup script authenticates the repository setup package checksum with a detached GPG signature and a release-pinned signer fingerprint before installing the repository configuration.
