## Summary

- Publishes Load Rift 0.2.17 with app metadata aligned across npm, Cargo, and Tauri packaging.
- Redesigns the workspace with a calmer dark-mode palette, stronger typography, and clearer visual hierarchy.
- Reduces workflow overwhelm by visually staging Source, Configure, and Run & Review instead of presenting every step with equal weight.
- Improves mobile and narrow-screen handling for actions, request rows, logs, monitor panels, result cards, and long collection content.
- Preserves existing import, request selection, runner configuration, smoke test, full test, monitor, export, validation, and persistence behavior.

## Linux APT Install

Enable the Load Rift APT repository once:

```bash
bash <(curl -fsSL https://github.com/ddv1982/load-rift/releases/latest/download/install-apt-repo.sh)
```

Then install or update normally:

```bash
sudo apt update
sudo apt install load-rift
```

The setup script authenticates the repository setup package checksum with a detached GPG signature and a release-pinned signer fingerprint before installing the repository configuration.

## Lore

Constraint: Release notes are read by the GitHub Actions release workflow from `docs/releases/v0.2.17.md` when available.
Constraint: The redesign must preserve the existing local k6 workflow and avoid re-opening backend or runner behavior.
Constraint: The workspace should feel less cluttered without hiding primary actions or breaking keyboard access.
Rejected: Showing Source, Configure, and Run & Review as equally loud panels | users need a staged operational path, not a wall of controls.
Rejected: Replacing the component stack or adding a UI framework | the existing React structure supports the behavior and only needed focused presentation changes.
Rejected: Claiming browser-verified polish | no Chromium, Chrome, or Firefox binary is available in this environment, so validation is automated plus static review.
Confidence: medium-high
Scope-risk: moderate
Directive: Ship the calmer progressive workspace as the current release line while keeping all load-test functionality intact.
