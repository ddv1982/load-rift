## Summary

- Clarifies the runner setup flow by separating the cURL extraction shortcut from manual Base URL and bearer token entry.
- Renames the cURL action to `Extract URL & Token` and shows clearer extraction feedback for users and assistive technologies.
- Avoids retaining pasted cURL commands that may contain bearer/JWT secrets by clearing the cURL input after extraction and no longer restoring legacy cURL drafts.
- Preserves existing manual Base URL or token values when a later pasted cURL command omits one of those fields.
- Bumps Load Rift metadata to version 0.2.12.

## Linux APT Install

Enable the Load Rift APT repository once:

```bash
bash <(curl -fsSL https://github.com/ddv1982/load-rift/releases/latest/download/install-apt-repo.sh)
```

Then install or update normally:

```bash
sudo apt update
sudo apt install load-rift
```

The setup script authenticates the repository setup package checksum with a detached GPG signature and a release-pinned signer fingerprint before installing the repository configuration.

## Lore

Constraint: Release notes are read by the GitHub Actions release workflow from `docs/releases/v0.2.12.md`.
Constraint: Pasted cURL commands can contain bearer/JWT secrets and should not be retained in browser storage or left visible after extraction.
Rejected: Persisting cURL drafts for convenience | it can retain authorization material despite auth fields intentionally not being restored.
Rejected: Clearing existing Base URL or token when a partial cURL omits that field | it makes the new manual-entry path unexpectedly destructive.
Rejected: Keeping the generic `Apply Curl` label | it did not explain which fields change or that extraction is a helper action.
Confidence: high
Scope-risk: narrow
Directive: Keep cURL import as a non-persistent extraction helper; do not reintroduce storage of pasted commands without redacting authorization material first.
