## Summary

- Refines the test workflow UI so run readiness, disabled actions, live metrics, final results, and export guidance are clearer without removing existing functionality.
- Makes runner inputs safer and easier to edit: invalid VU values are blocked, persisted invalid VU preferences fall back to defaults, and Base URL edits no longer trim while typing.
- Adds accessible validation feedback for runner and advanced JSON fields, and prevents malformed advanced JSON from reaching configuration validation, smoke tests, or load-test starts.
- Improves collection filtering and weighted-request guidance with clearer empty states, reset affordances, and responsive request-row polish.
- Bumps Load Rift metadata to version 0.2.11.

## Linux APT Install

Enable the Load Rift APT repository once:

```bash
bash <(curl -fsSL https://github.com/ddv1982/load-rift/releases/latest/download/install-apt-repo.sh)
```

Then install or update normally:

```bash
sudo apt update
sudo apt install load-rift
```

The setup script authenticates the repository setup package checksum with a detached GPG signature and a release-pinned signer fingerprint before installing the repository configuration.

## Lore

Constraint: Release notes are read by the GitHub Actions release workflow from `docs/releases/v0.2.11.md`.
Rejected: Keeping advanced JSON feedback advisory-only | it left invalid fields visually marked while backend validation and smoke tests could still receive malformed options.
Rejected: Trimming Base URL on every keystroke | it made manual editing feel jumpy and rewrote the user's in-progress input.
Rejected: Trusting persisted VU preferences without normalization | legacy or corrupted local storage could bypass the new local validation state.
Confidence: high
Scope-risk: moderate
Directive: Keep local readiness gates aligned with every backend-triggering action when adding new runner validation.
