## Summary

- Publishes Load Rift 0.2.15 with app metadata aligned across npm, Cargo, and Tauri packaging.
- Prevents stale collection import completions from overwriting newer imports, reset state, or picker errors.
- Prevents stale smoke-test completions from replacing newer smoke-test results or cleared smoke-test state.
- Adds regression coverage for stale async success and failure paths in collection import and smoke-test hooks.

## Linux APT Install

Enable the Load Rift APT repository once:

```bash
bash <(curl -fsSL https://github.com/ddv1982/load-rift/releases/latest/download/install-apt-repo.sh)
```

Then install or update normally:

```bash
sudo apt update
sudo apt install load-rift
```

The setup script authenticates the repository setup package checksum with a detached GPG signature and a release-pinned signer fingerprint before installing the repository configuration.

## Lore

Constraint: Release notes are read by the GitHub Actions release workflow from `docs/releases/v0.2.15.md`.
Constraint: Import and smoke-test actions may overlap when users retry quickly or clear state while backend work is still resolving.
Rejected: Cancelling frontend async work directly | the underlying Tauri calls do not expose cancellable requests, so stale completions are ignored with request sequencing instead.
Rejected: Leaving stale-result protection covered only by implementation review | the race behavior now has focused regression tests for success, failure, and clear/reset/error invalidation paths.
Rejected: Re-opening unrelated workspace behavior for this patch release | 0.2.15 is scoped to async race hardening and release metadata.
Confidence: high
Scope-risk: narrow
Directive: Keep the newest user action authoritative when import or smoke-test requests resolve out of order.
